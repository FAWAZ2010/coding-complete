Topic details

 

Object Data Structure:

  • The data in an object is stored in the value half of the key-value paring.
  • The key lets us access the value.
  • Syntax: var variablename = {key1:value1, key2: value2};
  • Example: var student = {name: Riya, age:8, grade:3, fav_color:pink};
  • Accessing way :Objectname.key i.e student.name or Objectname[key] i.e student[name]
  • If we try to access the key which is not in the arguments it returns undefined because there will be no matching key in an object. 

Math objects:

  • You have already learnt about built-in objects in JS, Math is one of the built-in objects of JS.
  • Example: Math.round()
  • In which Math is an object and .random() is a method.
  • This .round() method takes the decimal number and rounds down it to a nearest whole number. .floor() is a method from the builtin Math object.

Array:

  • Array is a special variable, which can store more than one value at a time.
  • Storing multiple values in a single variable is an array.
  • Syntax: var array_name = [value1, value2, .];
  • Example: var fruits = [Apple, Mango, Banana, Jackfruit]; 

 

Accessing Array:

  • Array elements can be accessed by referring to the index number.
  • Array index starts with 0.
  • [0] is the first element, [1] is the second element and so on.
  • For the above example:

 

fruits[0]

Apple

fruits[1]

Mango

fruits[2]

Banana

fruits[3]

Jackfruit

 

Array Methods:

  • Methods are actions that you can perform, we have already learnt about methods in JS.
  • Some of the in-built methods in the array are .join() and .pop().
  • .join() - joins all array elements into a string with a separator that is passed inside the parentheses.
  • Syntax: array_name.join(*);
  • Example: fruits.join(*) 
  • Result : Apple*Mango*Banana*Jackfruit
  • .pop() - pop methods remove the last element from the array.
  • Syntax: array_name.pop();
  • Example: fruits.pop();

 

Call Stack: 

  • Call stack is used by JS in order to keep the track of multiple function calls.
  • It follows the Last In First Out (LIFO) principle.
  • Example:

  • When the script runs the JS engine first places the global execution context denoted by main() function on the call stack().
  • Then the JS engine executes the average(10, 20); and creates an execution context for average() and pushes it to the top of the call stack.
  • Then the JS engine starts executing the average() function as it is in the top of the stack, as add() function is called in average() function, JS engine creates another function execution context for add() function and places it in the top of the stack.
  • So first the add() function will be executed. Then the average() function will be executed. Then the call stack will be empty, so it stops executing functions.
